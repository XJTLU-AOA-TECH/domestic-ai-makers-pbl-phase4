Phase 4 · Tactile Feedback · Tactile Interaction Reality

Tactile feedback integration and tactile interaction reality

触觉反馈集成，触觉交互现实

Students turn digital commands into physical feedback through screens, touch, lights, sound, vibration, servos, motors, wireless control, and intelligent car behavior.

Lecture introduction

中文课程简介

本课程主题是“触觉反馈集成，触觉交互现实”。学生把屏幕、触摸、灯光、蜂鸣器、震动、电机、舵机和小车运动整合起来，让数字指令变成可触摸、可感知、可执行的真实交互。

Students progress from touch menus and feedback outputs to wireless commands, motor/servo motion, remote cars, and intelligent pathfinding. The focus is turning interaction logic into physical, perceivable feedback and action.

Day 1

Screen layout and touch menu

Students design a screen layout and touch menu control. If no touchscreen is available, buttons can simulate menu input.

Topics

  • Screen layout
  • Touch feedback
  • Menu state
  • Short/long press fallback
  • AI screen layout prompts
Outcome: touch menu control prototype.
Day 2

Bluetooth/Wi-Fi light control

Students design wireless command formats and control lights through Bluetooth, with Wi-Fi as an extension.

Topics

  • Bluetooth commands
  • Wi-Fi web control
  • LED/light feedback
  • Communication data design
  • AI-assisted command design
Outcome: Bluetooth light control.
Day 3

Car motor and servo control

Students learn DC motor, motor driver, PWM, servo angle, and car motion functions.

Topics

  • DC motor
  • Motor driver
  • PWM speed
  • Servo control
  • Car motion program
Outcome: car motion program.
Day 4

Bluetooth remote car

Students control car motors through Bluetooth and design safe AI-assisted motor decisions.

Topics

  • Bluetooth command set
  • Car action functions
  • Safety stop
  • Wi-Fi extension
  • AI motor decision rules
Outcome: remote-controlled car.
Day 5

AI pathfinding car

Students map AI tags, sensor states, or classification labels to motor and servo responses for pathfinding.

Topics

  • AI tag / label
  • Manual/auto mode
  • Sensor-triggered control
  • Pathfinding rules
  • Final smart car demo
Outcome: intelligent pathfinding car.

Hardware checklist

  • ESP32-S3 / ESP32 development board
  • Separate touchscreen module for real touch input
  • LED or LED strip for light control
  • Bluetooth/Wi-Fi capable ESP32 board
  • Two-wheel car chassis, DC motors, motor driver
  • Servo motor
  • Ultrasonic sensor for pathfinding
  • Battery pack or external motor power

Software checklist

  • Arduino IDE installed
  • ESP32 board package installed
  • Serial Monitor works at 115200 baud
  • BluetoothSerial or BLE support for selected board
  • Wi-Fi web server examples available
  • No real Wi-Fi credentials committed to GitHub

How to use this folder

如何使用本文件夹

  1. Open this index.html first to understand the course structure.
  2. Read README.md for the overall learning path.
  3. Go to each day folder and follow the lab README files.
  4. Open each .ino file in Arduino IDE.
  5. Test each module alone before combining it into the car project.
  6. Submit final code, lab report, and evidence files in the outputs/ folder.
