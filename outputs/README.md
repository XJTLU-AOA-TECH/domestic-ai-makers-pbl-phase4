# Student Outputs

Students submit lab results in this folder.

## Folder structure

```text
outputs/
├─ lesson1/
├─ lesson2/
├─ lesson3/
├─ lesson4/
├─ lesson5/
└─ templates/
```

## Recommended submission format

Each student or team can create a folder using this format:

```text
outputs/lesson1/team-01/
outputs/lesson2/team-01/
outputs/lesson3/team-01/
outputs/lesson4/team-01/
outputs/lesson5/team-01/
```

## Expected outputs

| Lesson | Output |
| --- | --- |
| Lesson 1 | Touch menu control prototype |
| Lesson 2 | Bluetooth light control |
| Lesson 3 | Car motion program |
| Lesson 4 | Remote-controlled car |
| Lesson 5 | Intelligent pathfinding car |

## What to submit

For each lab, students should submit:

1. Final Arduino `.ino` code.
2. A short lab report using the template in `outputs/templates/lab-report-template.md`.
3. Photos or screenshots if available.
4. Debugging notes: what failed and how it was fixed.
5. For wireless labs, do not include real Wi-Fi passwords or private network information.

## Suggested file names

```text
final_code.ino
lab_report.md
photo_01.jpg
serial_monitor_screenshot.png
web_control_screenshot.png
project_demo_notes.md
```

## Important

Do not submit private API keys, Wi-Fi passwords, personal tokens, private photos, or other secrets.

## 中文说明

`outputs/` 文件夹用于学生提交实验结果。建议每个学生或小组按照课次建立自己的提交文件夹，例如 `outputs/lesson1/team-01/`。

本阶段的主要产出依次为：触摸菜单控制、蓝牙灯光控制、小车运动程序、遥控小车、智能寻路小车。

每次提交应包含最终 Arduino 代码、实验报告、照片或截图、调试记录。无线控制实验不要提交真实 Wi-Fi 密码、个人热点密码或其他敏感信息。
