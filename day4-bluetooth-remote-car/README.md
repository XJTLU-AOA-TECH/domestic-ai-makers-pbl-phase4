# Day 4 — Bluetooth Remote Car and AI-Assisted Motor Decisions

## Goal

Students build a remote-controlled car and use AI-assisted thinking to design motor decision rules.

By the end of Day 4, students should be able to:

1. Use Bluetooth commands to control car motion.
2. Use Wi-Fi web control as an optional extension.
3. Design command formats for car control.
4. Add safety stop behavior.
5. Explain how AI decision rules can map input states to motor actions.

## Labs

| Lab | Description |
| --- | --- |
| `01-single-motor-test` | Test one motor direction and speed |
| `02-dual-motor-actions` | Build basic car movement functions |
| `03-bluetooth-car-control` | Use Bluetooth commands to control the car |
| `04-wifi-web-car-control` | Use a phone browser to control the car through Wi-Fi |

## Key concepts

- Remote control: user sends wireless commands to the car.
- Bluetooth command set: `F`, `B`, `L`, `R`, `S`.
- Wi-Fi web control: browser-based control panel.
- AI-assisted decision design: ask AI to propose safe motor responses.
- Safety stop: every remote control project needs a reliable stop command.

## AI prompt examples

```text
Design safe motor control rules for a Bluetooth remote car using F, B, L, R, and S commands.
```

```text
Suggest how to reduce speed when sensor data says the car is close to an obstacle.
```

## Suggested classroom flow

1. 20 min — Remote car command design.
2. 30 min — Review motor action functions.
3. 40 min — Bluetooth remote car test.
4. 35 min — AI-assisted motor decision rules.
5. 35 min — Wi-Fi web control extension or team demo.

## Safety notes

- Test with wheels lifted first.
- Keep speed low.
- Always test the stop command.
- Do not run the car near people or fragile objects.
