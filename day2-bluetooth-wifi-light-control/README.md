# Day 2 — Bluetooth/Wi-Fi Communication and Light Control

## Goal

Students learn Bluetooth and Wi-Fi communication by controlling lights from a phone or computer.

By the end of Day 2, students should be able to:

1. Explain how wireless commands control hardware.
2. Design simple command formats such as `ON`, `OFF`, or color names.
3. Use LED, buzzer, or vibration feedback to show command results.
4. Build a Bluetooth light control prototype.
5. Use Wi-Fi control as a standard or advanced extension.

## Labs

| Lab | Description |
| --- | --- |
| `01-led-feedback` | Use LED patterns to show device state |
| `02-buzzer-feedback` | Use buzzer tones as feedback |
| `03-vibration-feedback` | Use vibration motor as haptic feedback |
| `04-servo-feedback` | Optional motion feedback |
| `05-bluetooth-light-control` | Control a light through Bluetooth commands |
| `06-wifi-light-control` | Control a light from a Wi-Fi web page |

## Key concepts

- Wireless command: data sent from a phone/computer to the board.
- Bluetooth control: simple local wireless command input.
- Wi-Fi web control: browser-based interaction over a network.
- Communication protocol: a shared rule for command names and responses.
- AI-assisted communication design: ask AI to design command formats and response messages.

## AI prompt examples

```text
Design a simple Bluetooth command set for a light with ON, OFF, RED, BLUE, and BLINK modes.
```

```text
Help me design a Wi-Fi web page with buttons for controlling LED colors.
```

## Suggested classroom flow

1. 20 min — Bluetooth/Wi-Fi concepts and command design.
2. 25 min — LED feedback review.
3. 35 min — Bluetooth light control.
4. 35 min — Wi-Fi light control extension.
5. 25 min — AI-assisted communication design and testing.
